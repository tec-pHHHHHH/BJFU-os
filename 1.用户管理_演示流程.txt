全程命令流程:

一、用户管理

1.2注册-异常校验

首先输入register user 123456，能够正常登录。
·测试重复用户名：然后再次输入register user 123456，显示已存在。
·测试非法字符：输入register user@ 123456
·测试空密码：register user

1.3登录-正常 + 异常
·正常登录：login user 123456
·错误密码：login user 11111，失败
·不存在的用户：login userr 123456，没找到

1.4密码哈希 + 3次锁定

ppt要求哈希不可逆存储，检查user.db中不含明文密码

hexdump -C data/users.db | grep 123456
(这个先不管，好像运行不了)

下面是3次锁定
login user 1输入三次，第三次应该显示账户已被锁定，再次输入正确密码，也发现依然显示账户锁定。

exit退出进程，再次运行后尝试登录，发现仍然锁定，说明是保存在内存中的，不是一次性。
